> 核心: gate I の承認を「ローカル commit」から「**push + PR 作成**」へ拡張し、人間レビューを GitHub の標準フローに接続する。
> PR の作成有無は state.json で管理し、gh の照会には依存しない（決定的・再入可能）。

## 1. git セクションの拡張

- `mode` の許容値に `"pr"` を追加。`pr` の `ensure_flow!` の挙動は `branch` と同一（ブランチ確立まで）。
- 新キー（mode pr のときのみ意味を持つ）:
  - `"remote"` 既定 `"origin"`
  - `"gh_cmd"` 既定 `"gh"`（テストではスタブスクリプトの絶対 path を渡す）
- `load!` 検証: mode pr のとき `gh_cmd` が実行可能であること（`System.find_executable` または絶対 path の存在＋実行権限）。
  不可なら `Mix.raise`。struct に `git_remote` / `git_gh_cmd` を追加。

## 2. `Workspace.push_and_create_pr!(ws, branch, title, body, create_pr?)`

- `git push -u <remote> <branch>` を実行（remote 不在・失敗は `Mix.raise`、メッセージに remote 名を含める）。
- `create_pr?` が true のとき `gh_cmd` で `["pr", "create", "--base", <git_base>, "--head", <branch>, "--title", title, "--body", body]`
  を実行し（cd: ws.path）、**stdout の最終非空行を PR URL として返す** `{:ok, url}`。false のときは push のみで `{:ok, nil}`。
- gh 失敗（非0 exit）は `Mix.raise`（出力先頭 300 字を含める）。

## 3. dev タスク統合（apply 時に PR、state で一度きり管理）

- implement の apply 成功後（`resume_implement` / `await_implement` の `{:ok, _sha}` 分岐）、mode が `:pr` なら:
  1. branch 名を解決（ensure_flow! と同じ template/slug 規則。`Workspace.branch_name/2` 相当を公開してよい）。
  2. `create_pr?` = state に `"pr_url"` が**無い**こと。
  3. title = `tracefield: <issue.md 先頭行 60字>`、body = 決定的なテキスト:
     `issue: <issue dir basename>\nchanges: <change ids>\ndecisions: <承認済み decision ids>\npolicy: <policy entry text>\n\nGenerated by tracefield pipeline`。
  4. `push_and_create_pr!` 実行。URL が返れば state に `"pr_url"` を保存し `PR: <url>` を表示。
     2回目以降（QA 差し戻し後の再 apply）は push のみで、表示は `PR 更新: push 済み（<pr_url>）`。
- `print_implement_done` / `print_qa_done`: state に `pr_url` があれば `PR: <url>` を1行表示
  （state を読むため必要なら引数を追加してよい。--status でも表示）。
- mode pr 以外の挙動は brief-31 と完全に同一（既存テスト無変更で green）。

## 4. テスト（完全オフライン。`test/workspace_test.exs` / `test/dev_task_test.exs` 追記）

- **スタブ方式**: tmp dir に実行可能スクリプト `fake-gh` を書く —— 受け取った引数を `gh-args.log` に追記し、
  `https://example.com/pr/1` を stdout に出して exit 0。失敗系スタブは exit 1。
  remote は `git init --bare` した tmp bare repo を `git remote add origin` で接続（push はローカルで完結）。
- unit: mode pr の load! 検証（gh_cmd 不在で raise）。push_and_create_pr!（push 後に bare 側へ branch が存在 /
  create_pr? true で URL が返り gh-args.log に期待引数 / false で gh 未呼び出し / gh exit 1 で raise）。
- e2e（mock organ、mode pr）: implement 承認 → bare origin に branch が push され、gh が1回だけ呼ばれ、
  state.json に pr_url が入り、出力に `PR: https://example.com/pr/1`。
  qa fail（test_cmd "false"）→ 差し戻し → 再承認 → **push は再実行されるが gh pr create は呼ばれない**
  （gh-args.log の create が1回のまま）→ test_cmd を true に直して qa pass → done 出力に PR URL。
- 互換: 既存の current/branch/worktree e2e が無変更で green。
