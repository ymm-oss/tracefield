# 概要 — 半溶解性オーケストレーションと「統治可能な探索」

> 本書は `tracefield` プロジェクトの概念的背景を述べる。実験設計の詳細は [`experiment-plan.md`](./experiment-plan.md)、
> 用語は [`glossary.md`](./glossary.md) を参照。

## 0. なぜ"半"溶解か — 中核概念

半溶解性の出発点は、**人間とAIの「開放性」の違い**にある。

- **人間は大部分"閉じた"存在**: 記憶を直接共有できず、自我と境界を防衛する。だから人間同士の協働は**言語という狭く損失の多い帯域**（直列化→発話→再解釈）を通すしかない。
- **AIは"開いた"存在**: メモリや状態を共有でき、自身の存在を防衛しない。だからAI同士は**状態そのものを溶かし合える** → 言語のボトルネックを越えて、**人間同士より深く・高効率・高精度に繋がれる**。

ここに半溶解性の核心価値がある: **AIの開放性を使って、人間の集団を超える協働を実現する。**

ただし ── **完全に溶け合うと、各エージェントの偏り（視点・専門性・オリジナリティ）が均質化して消える。** すると複数で探索する意味（多様な死角への到達）が失われる。

> ゆえに **各自の偏りを温存する程度の "半" 溶解に留める。** 深く共有して高効率に繋がりつつ、オリジナリティは保つ。"半" はこのトレードオフ上の動作点。

### 中核価値の二面
1. **創発価値（攻め）**: 開放性による深い協働 ＋ **多様性（偏り）の温存**。半溶解性の本来の目的。
2. **統治価値（守り）**: 溶けた寄与を後から **追跡・隔離・撤回** できること。完全融合でなく "半" だからこそ可能。

**本実験計画（§1 以降）が主に検証するのは 2（守り）**。1（攻め＝協働の質と多様性保存）は、操作的には §9.3 Exploration Retention（統治を強めた時に探索の多様性がどれだけ死ぬか）として現れるが、検証は手薄。

### 計画書内の操作化との対応
| 中核概念 | 計画書の操作化 |
| --- | --- |
| 偏り（オリジナリティ）の温存 | §9.3 Exploration Retention / Field Actor の sensitivity profile |
| 溶け込み（部分融合） | Absorption log / projection |
| 完全融合の不可逆性 | §12「取り込み時に不可逆的融合の可能性を説明」 |
| "半"（部分可逆） | provenance / reversibility（追跡・撤回機構） |

> 注: この §0 の概念定義はプロジェクト本来の核であり、初期実験計画書には**明文化されていなかった**（名前・語彙・上記指標がその影を落としていた）。2026-06-10 に言語化され本書に記録。

## 1. 何を問うているのか（統治の側面）

マルチエージェント探索には、よく知られたトレードオフがある。

- **自由形式探索**は開放的で、誰も所有していない「介在的な死角」まで踏み込める。しかし、いったん**汚染・虚偽・撤回された入力**が混ざると、その影響がどこまで波及したかを追跡できず、隔離も撤回も難しい。
- **固定Roleパイプライン**は統治しやすい。誰が何を担当し、どこで止めるかが明確だ。しかし探索が硬く、単一Roleの担当範囲をまたぐ問題（複数領域の相互作用でのみ顕在化する死角）を取りこぼす。

`tracefield` が検証する **半溶解性オーケストレーション** は、この二項対立に対する設計仮説である。

> 開放的探索を保ちながら、**provenance（来歴）・reversibility（可逆性）・gateability（ゲート可能性）** を維持できるのではないか。

| 様式 | 開放性 | 統治可能性 |
| --- | --- | --- |
| 自由形式探索 | ◎ | ✕（追跡・隔離・撤回が困難） |
| 固定Roleパイプライン | ✕（硬く、介在的死角を見落とす） | ◎ |
| 半溶解性オーケストレーション | ◎ を保つ | provenance / reversibility / gateability を維持 |

## 2. 中核価値は「死角発見」ではなく「統治可能な探索」

この点が本プロジェクトの主張の肝である。

「より多くの死角を見つけられる」だけなら、エージェントを増やせばよい。本当に問いたいのは次である。

> 汚染・虚偽・撤回された入力が混ざったとき、その**下流影響をどれだけ正確に特定し、隔離・切除・再評価できるか**。

したがって主アウトカムは **Impact Recall / Precision**（影響特定の再現率・適合率）であり、死角の数は副次的指標に過ぎない。

## 3. EGI — 創発層とガバナンス層の双方向インターフェース

半溶解性オーケストレーションは、自由な創発層（exploration layer）と、コミットを管理する governance layer を**双方向**でつなぐ。この接続が **EGI**（創発−ガバナンス間インターフェース／名称は推定）である。

```
            governance layer
                 ▲   │
   Frame         │   │   Delta
   Revision      │   ▼   Packaging
   Trigger (上り) │   │   (下り)
                 │   ▼
            emergent / exploration layer
```

- **下り（Delta Packaging）**: 創発層の生の痕跡を、PCE gate が評価できる candidate delta に整形する。整形しすぎて死角の実質を洗い流すリスクが **Packaging Loss**。
- **上り（Frame Revision Trigger）**: 既存の frame では表現できない発見を上位へ突き上げる。ただし創発層が frame 改訂権限を**勝手に行使しない**ことが要件。

両方向が「実質的に機能するか／重要な意味を失わないか」が EGI の評価軸になる。

## 4. なぜ接地真実が難しいのか

影響追跡の性能を測るには「真に影響を受けた集合」が要る。だが **provenance log をそのまま正解にしてはいけない** — それはシステムの自己申告であって、実際の因果ではない。

そこで本実験は **反実仮想再実行（counterfactual re-run）** を一次の接地真実に用いる。汚染入力を「含む条件」と「除去/訂正した条件」を可能な限り同条件で走らせ、出力の差分を「影響を受けた集合」とみなす。これに**ブラインド専門家裁定**を補助的に重ねる。

ただし反実仮想は単一入力の効果には強いが、複数入力の相互作用・遅延・経路変化は完全には捉えられない。これは限界として明記する。

---

## 5. Provenance / Consent の正直さ

本プロジェクトは過剰な約束をしない。

- **Append-only provenance** は「改竄されていないこと」を保証するが、「最初の記録が真であること」は保証しない。よって *改竄不能な虚偽記録* になりうる。真正性問題は本実験では完全には解かず、provenance が影響追跡・隔離に**どれだけ寄与するか**を測るに留める。
- **Consent / Withdrawal** の重心は「完全撤回の約束」ではなく、**informed consent + best-effort withdrawal + irreversibility disclosure** に置く。完全には撤回できない場合、その限界を正直に開示できるかを評価する。

## 6. この実験で結論づけられること

本実験は、半溶解性オーケストレーションが —

- **「よくできたオントロジー（概念整理）」に留まる**のか、
- それとも **「統治可能な探索」の実装可能な設計仮説として成立する**のか、

を切り分けるための**最小実験**である。

- 肯定なら、自由形式探索と固定パイプラインの間にある**実在する設計空白**を埋める候補になる。
- 否定なら、哲学的には整合していても実装上は成立しない可能性が高い。

具体的な成功/失敗基準は [`experiment-plan.md` §15–§16](./experiment-plan.md#15-成功基準) を参照。
