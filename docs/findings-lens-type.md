# レンズ種別 知見 — ロール/フレームワーク/哲学/論理操作の比較と合成ボトルネック

> 「エージェントをロールでなく止揚等の哲学概念や分析フレームワークで作ると効果があるか」を
> tracefield 上で検証した結果。日付: 2026-06-18。モデル: `claude-sonnet-4-6`（CLIアダプタ）。
> 生 run は `runs/lens-exp*.report.md`（`runs/` は gitignore のためローカルのみ）。

## 0. 結論（要旨）

> §6.6（確証実験）で 1 と 3 は精査され、過大主張を訂正済み。以下は**訂正後**の確定版。

1. **直交する複数の哲学分野は、リッチなロールより「直交した考慮・死角」を多く表面化する**（盲検 LLM 審判で確証、§6.6）。※当初の「哲学が別の答えを出す」は決定レベルでは再現せず撤回。効くのは「哲学」でなく分野間の**直交性**。構造変更型論理操作（場合分け）も同様に有効。
2. **「操作」はレンズにできない。** 止揚（合成）も反証（批判）も自前のテーゼを持たず他レンズの出力に作用する＝**ステージ**に置くべき。
3. **一枚岩 LLM 合成の忠実性は入力規模で劣化する（弱いモデルほど激しい）。intrinsic な confabulation ではない**（小入力では忠実、§6.6）。対策は**敵対項目を1アクターに隔離＝全 LLM 呼び出しを忠実な小規模域に留め、統合は機械的集約**。tracefield の stage / actor / `per_input` / ReferenceStore / retract がこれを表現できる。

## 1. 実験設計（再現条件）

- 経路: 新CLI `tracefield run`（旧 `consult` は廃止）。`flow.toml` の `stages.<id>.actors.roles` に `agents.json` の id を並べると、その agent の lens で actor が駆動。
- 入力連結: `inputs=["stage:<id>"]` で前段エントリを受ける。複数ラウンドは `[long_run] cycles=N cycle_stages=[..]`。反証1件=1アクターは `actors.mode="per_input"`（入力エントリを actor にシャード）。
- adapter=cli, command="claude", model=claude-sonnet-4-6。
  - 補足: ローカル ollama 31B は 300s/req タイムアウト、e4b は先行エージェントを逐語エコーするため不適。`~/.cargo/bin/tracefield` は古く claude アダプタに `--force` を渡し壊れている。`./target/release/tracefield` を使う。
- 題材: 「レガシー決済モジュールをフルリライト(A)/段階改修(B)/現状維持(C)」（決定的題材）と「顧客行動データの第三者収益化 A/B/C」（価値含み題材）。task.md を共有し、変数を lens 定義のみに限定。

## 2. レンズ種別の比較（実験1相当）

同一題材を 4 パネルで議論させた（各4レンズ）。全レンズが結論 B に収束したが、**分化の質**が決定的に異なる。

| パネル | 分化の性質 | 例 |
|---|---|---|
| ロール（BE/PM/SRE/FIN） | 語彙違いの**冗長**な分化 | 同じ事実を同じ論理で再述 |
| フレームワーク（RISK/ECON/TOC/REVERS） | 固有の分析操作による**非自明な再framing** | 「Cは最安でなく最も高価」「時限式の一方通行ドア」「他は症状、制約は1つ」 |
| 哲学（UTIL/DEONT/PHENO/GENEA） | 最も直交した観点で**死角を照射** | 「恐怖が現状維持バイアスを強化」「同じ組織パターンの再生産に注意」「義務と暗黙の約束」 |

最良の単一洞察はすべて非ロールレンズ発。

## 3. 結論分岐とラウンド（実験3相当）

決定的題材を rounds=2（相互可視・相互反論）で実行。

- **答えレベルの分岐を生んだのは哲学レンズだけ**: UTIL（功利主義）のみ c1 で A を選択（他は B）。
- rounds=2 で UTIL は「条件付きB」へ**弁証法的に移動**（P(B完遂)が示せなければ A に戻る、と退路を保存）。DEONT が GENEA のゲーティングを「義務要件」に昇格させる等、レンズ同士が論拠を供給し合った。
- sonnet では「悪いエコー（逐語コピー）」は起きず、収束は相互引用を伴う建設的なもの。エコーは e4b 固有のモデル能力問題。
- 価値含み題材でも全レンズ B に収束。**支配的な妥協案がある題材では結論は割れず**、差は正当化と条件付きに出る。

## 4. 論理操作レンズ（実験A相当）

- **生成系は一枚岩でない**。判別基準は「操作が出力の SHAPE を変えるか」。
  - 場合分け(CASES): 決定変数で8ケースに分解し C が有効な 1/8 を特定。**全実験で唯一 C の成立条件を発見**。答えの形を「1つ選ぶ」→「条件マップ」に変える最強の論理操作レンズ。
  - 演繹(DEDUCT): 射程内に留まり「C は必然除外・A/B は未決定」と断定を拒否＝規律ある独自貢献。
  - 帰納/アブダクション: 既定推論の語り直しで冗長。
- **批判系（反証 FALSIFY / 反例 COUNTER）はステージで最高の決定的内容**を産出（「B は遅延する C に過ぎない」「再識別リスクは A/B 同一で"Bは倫理的に清潔"の前提が崩壊」「収益不足→倒産時の従業員80名という未評価当事者」）。レンズ（テーゼ生成）でなく verify ステージが正しい居場所。

## 5. 合成ボトルネックと解決（実験B/構造化/審判）

止揚（合成）ステージを 3 段階で検証した。

1. **素の synthesis**: 6レンズ中 PHENO（最も直交）を「全5レンズ」と数え落とし、反証を吸収せず多数決を追認。
2. **構造化 synthesis**（全レンズid列挙必須＋各反証にverdict必須＋トークン3000）: PHENO 脱落は解消したが、**反証点検表を confabulate**（verify に無い項目を流用、反証を支持論に反転、最強の反証を黙殺）。→ **フォーマット遵守 ≠ 実質吟味。見かけの頑健性を製造する分、無構造の素通りより危険。**
3. **審判アーキテクチャ**（`adjudication` ステージ `mode=per_input roles=["ADJ"]`：反証1件=1独立アクター）: **決定的成功**。6反証すべてに独立 verdict、脱落・矮小化・反転ゼロ。構造化で黙殺された最強反証（オプトイン打診自体が信頼毀損）も正面から審判（却下1＋条件付き維持5）。複数 ADJ が独立に「反証の含意"→A"は誤り、代替は C」と同じ補正を発見。結論 B は不変だが「見せかけの頑健性」→「5条件付きで反証に耐えた B」へ質的転換。
   - ただし**最終の一枚岩 SYNTH は審判結果を入力されてもなお confabulate**（「コンテキスト切断により確認不可」と述べ弱い3項目を再捏造）。

→ **真の成果物は per-refutation の ADJ 判定（durable / auditable な entry）。最終結論は LLM 再合成でなく機械的集約で導くべき**（どの ADJ も「覆る」を出さない → B 存続＋条件の和集合）。これが ReferenceStore / provenance / retract の正しい使い所。

## 6. 設計原則（持ち帰り）

- エージェント定義は2軸で判定する: **(a) 偏り(divergent bias)を持つか**（価値構造 > 構造変更型論理操作 > ドメイン枠 > 冗長系）、**(b) 他者の出力に作用するか**（合成=止揚 / 批判=反証 はレンズでなくステージ）。
- パネルに**対立する哲学レンズ（功利主義⇄義務論等）を1〜2枚混ぜる**と、ロールだけのパネルが見落とす代替案と価値次元が表面化する。
- **合成・批判ステージは敵対項目を1アクターに隔離し、最終結論は機械的集約で出す**。一枚岩 LLM 合成に丸投げしない。

## 6.5 機械的集約の実装（confabボトルネックの解消）

工程6（機械的集約）を `tracefield aggregate --store <jsonl> [--stage adjudication]` として実装した
（`crates/tracefield-cli/src/main.rs`）。durable JSONL ストアの adjudication ステージの decision エントリを読み、
各 verdict を**明示的な「判定:」ラベルの先頭だけ**で {overturn / conditional / reject / maintain / unclassified}
に決定論的分類（プロセ中の「覆す」「維持」に汚染されないよう head 24 文字に限定）、

- overturn が1件でもあれば結論 changed、
- unclassified があれば indeterminate（清浄な fold を阻む＝要対応として surface、silent drop なし）、
- それ以外は maintained ＋ conditional verdict の和集合を条件として列挙。

既存の審判 run（`runs/lens-exp-synth-adjudicate.jsonl`）で検証: 6 verdict → conditional 5 / reject 1 /
overturn 0 → **maintained(B) ＋ 5 条件**を全 id 付きで出力。LLM SYNTH が起こした PHENO 脱落・反証捏造・
コンテキスト切り詰めは構造的に発生しない。分類のプロセ汚染耐性はユニットテストで固定（`classify_verdict`）。

## 6.6 確証実験（2026-06-18 追補）— 2つの load-bearing 主張を対照＋盲検で精査

自己批判で挙げた交絡を、対照実験で殴った結果、**2つの headline は過大で、訂正版が確証された**。

### 主張A: 「哲学 > ロール」— 決定分岐は撤回、直交考慮は盲検で確証

- **撤回**: 「哲学は別の答え(A/B/C)を出す」は単発で再現せず（role/phil × terse/rich の 2×2、全アーム全 B 収束）。元の UTIL→A は rounds=2 の 1 事例のみで、決定レベルの主張としては過大だった。
- **richness 交絡は否定**: role-rich（死角申告つきの richな desc）でも結論は全 B、かつ**「共有事実への職能的重点」止まり**で構造的再framing を出さない。差は記述量でなくレンズ型由来。
- **確証された本来の主張**: 「**相互に直交する複数の哲学分野**（帰結主義⇄義務論⇄現象学⇄系譜学）は、リッチなロールより**直交した考慮・死角**を多く表面化する」。匿名2パネルを**盲検 LLM 審判にラベル入替で2回**かけた結果、**両方向とも哲学パネルを選択**（固有の直交考慮 哲学10 vs ロール7）。審判は独立に「ロール固有＝同一事実平面の定量精度向上（MTTR倍率・制裁金額）／哲学固有＝財務・技術と直交する価値次元（義務論・現象学・組織史）」と articulate。**効くのは「哲学」でなく分野間の直交性**。

### 主張B: 「合成は confabulate する」— intrinsic は撤回、スケール×モデル効果と確証

- **撤回**: 「一枚岩 LLM 合成は本質的に confabulate する」は過大。**極小入力(3レンズ)では素の SYNTH は 3/3 忠実**に全レンズ列挙＋反証統合（脱落・捏造ゼロ）。
- **確証された訂正版**: 合成忠実性は**入力規模で劣化し、弱いモデルほど激しい**。SYNTH 自身の自己カウントが客観証拠: 小(3)→「3レンズ」正、大(6)→sonnet「全**5**レンズ」(BE脱落)、大(6)→haiku「**Three** lenses converge on **A**」(6→3誤カウント＋英語化＋結論反転)。
- → **アーキテクチャ修正(per-refutation 隔離＋機械的集約)が効く因果が明確化**: 「LLM は信用ならない」でなく「**全 LLM 呼び出しを忠実な小規模文脈域に留め**、規模を要する統合だけ機械集約に逃がす」。§6.5 の機械的集約はこの小規模域維持装置。
