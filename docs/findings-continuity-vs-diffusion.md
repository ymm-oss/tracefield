# 連続性 vs 拡散 知見 — narrow-answer タスクで単一強モデルに勝つには、観点でなく文脈の隔離

> 「実応用の narrow-answer 戦略タスクで、tracefield のパネルは単一強モデルに勝てるか。
> 勝てるなら勝因は『観点』か『文脈の隔離』か」を検証した結果。
> 題材は**匿名化した実タスク**（大規模専門サービス組織内での開発ツール普及戦略。組織名・固有事実は秘匿）。
> 日付: 2026-06-19。モデル: **`codex-app-server`** / 単一比較は `codex exec`。審判は新規 codex セッション（出自盲検）。
> 姉妹ドキュメント [findings-lens-type.md](./findings-lens-type.md) / [findings-diffusion-thinking.md](./findings-diffusion-thinking.md)。生 run はローカル（匿名化・`runs/` は gitignore）。

## 0. 結論（要旨）

1. **narrow-answer タスクでは、直交レンズのパネルは単一強モデルに answer-quality で並ばれる。** 答えの空間が狭い問い（外的制約が答えをほぼ一意に縛る戦略問題）では複数の直交レンズが収束し、多様性の保険が空振る。単一モデル1パスが同等以上の具体性を出した。[findings-lens-type.md](./findings-lens-type.md) §2-3 の「decision は収束するが分化の質が違う」を実タスクで延長すると、**質の違いが answer 優位に転化しない領域がある**。
2. **真の設計軸は「単一 vs マルチ」でなく「連続性(continuity) vs 拡散(diffusion)」。** 単一＝連続性の機械（1文脈で推論が累積・深い／弱点は早すぎる整合化で*分岐*を見逃す）。パネル＝拡散の機械（隔離文脈に散らし発散・網羅は強いが累積的深さを失い、フレーム共有だと再収束）。[findings-diffusion-thinking.md](./findings-diffusion-thinking.md) の Diffusion 直感を対立物（連続性）と対にして定式化。
3. **勝ち筋は「拡散 → 連続性の交互織り」。** 発散で分岐・死角・overturn を見つけ、その**選ばれた分岐を単一の連続した深い1パスに渡す**と、単一1パスを盲検で上回った。
4. **勝因は「観点」でなく「文脈の隔離」。** 単一に**同じレンズのシステムプロンプトを逐語で渡し1文脈で全工程を演じさせる**と、観点なしの素の単一より**悪化した**（後述 W）。価値はレンズ内容（プロンプトで移植可能）でなく、(a) 各立場を独立文脈で発展 (b) 各反証を per_input で独立審判 (c) 深い合成を専用の別パスに分離——という**構造的隔離**。これは flow-design 鉄則2「一枚岩に多レンズ＋反証を渡すと規模で劣化」を強モデルで実証し、[findings-diffusion-thinking.md] の「collapse は反復でなく単一の心への集約の性質」と一致。
5. **代替不能な非対称は「稀な構造修正を独力で出せるか」に局在する（§6 で多段に自己反証）。** 単一は装飾でなく本物の決定ゲート（選定基準・go/no-go 閾値・価格論理・kill 条件）を量産でき、当初 decision-relevant rubric では単一が勝った。だが交互織りの連続性パスに**「決定ゲートを出し尽くせ」を明示**すると、同 rubric で交互織りが 46 対 40 と反転勝利（approval-first 順序＝de-risk を保持したまま密度でも並ぶ）。→ **交互織りは両次元（ゲート密度＋稀な構造修正）の和を取れ、単一は構造修正を独力で出せず片方止まり。** ゲート密度の差はプロンプト抜けで埋まるが、安全性/承認の順序の overturn は単一が5審判通して出せなかった——ここが構造的・代替不能。

## 1. 実験1: 直交レンズ審議は単一に並ばれる

正準骨格（framing → 直交5レンズ → verify[FALSIFY/COUNTER] → adjudication[per_input] → `aggregate`）。aggregate=maintained/overturn=0/条件6。5レンズは独立に同一骨格へ収束。同一 task を単一1パスに渡すと、同じ立て直し＋反証に到達し**より具体的**だった。→ この問いではパネルの固有価値は answer でなく**監査価値**（provenance/分離条件/no-silent-drop）。[findings-governance-vs-fusion.md] と整合。

## 2. 実験2: 立場トーナメントで拡散を人為点火

両立しない5つの戦略コミットメント（資金源・権限・導入経路が相互排他）に**コミットさせて各々完全な実行計画**を書かせる → JUDGE が隔離採点(per_input) → 赤チーム[FALSIFY/COUNTER]が全計画を破壊 → ADJ が反証1件=1審判＋**反証プローブ**（投資前に前提を潰す最小実験）→ synth。

- 計画は**収束しなかった**（立場コミットメントの強制が再収束を防ぐ）。aggregate=**changed/overturn=1**。
- 覆した反証＝**安全性/承認ゲートの順序**: 単一が断定で素通りした「外部データ利用・承認境界の確認を*先頭*に置く」を、独立赤チームが強制した。

## 3. 実験3: 拡散 → 連続性の交互織りが単一1パスを盲検で上回る

実験2の**勝ち分岐**（背骨＋接ぎ木＋生き残った条件＋反証プローブ＋overturn の順序修正）を新規モデルに渡して**「方向は再議論せず一本の連続思考で深掘りせよ」**と指示（連続性パス）。盲検審判で **46/50 対 37/50** で単一1パスに勝利。深さは連続性パスが、分岐・順序修正・深掘りブリーフは拡散ステージが供給。tracefield 単体（短い synth）でも単一単体でも届かない。

## 4. タイブレーク: 交絡（compute／ブリーフ／観点）を消しても隔離が効く

§3 の優位には交絡があった——compute、プロンプト非対称（枠を渡した）、観点。3条件の公平版を作り全て盲検で比較した。

| 単一に何を足したか | 盲検スコア（/50） |
|---|---|
| X: 開いた課題のみ(34k tok) | 36–37 |
| Z: ＋成果物の9項目の枠＋64k tok＋web検索（**観点は渡さず**、潰す順序は自分で決めさせる） | 45–46 |
| **W: ＋レンズのシステムプロンプトを逐語＋1文脈で全工程（5立場→採点→赤チーム→審判→合成）** | **40** |
| **Y: tracefield（隔離立場→独立審判→専用の連続深掘りパス）** | **47–48** |

読み取り（全て出自盲検・同一ルーブリック）:
- **構造/枠は単一を大きく底上げ**（X→Z, +約10点）。「枠を書けること」自体は価値だが、それは単一でも持てる。
- **だが同じ観点を1文脈に詰め込むと劣化**（Z→W, −約6点）。W は5計画＋反証で文脈が埋まり、**最終合成が圧縮され de-risk（kill条件・失敗時縮退・潰す順序）が最初に崩れた**（審判: W の de-risk=7 vs Y=10, Z=9）。発散・批判・合成は同一文脈で予算を奪い合う。
- **隔離した Y が最高**。ただし Y↔Z 差は1〜3点で**run間分散内**＝answer スコアでは互角圏。Y の頑健な edge は両審判とも「特記＝投資前に潰す前提の*順序*」で safety/approval を先頭に置いた点を健全と評価したこと。

→ **代替不能なのは「観点」でも compute でもなく「文脈の構造的隔離」**：独立した発散・per_input の独立審判・発散と切り離した専用の深掘り。単一でこれを1文脈に畳むと、最も価値ある de-risk の厳密さが最初に失われる。

## 5. tracefield への設計フィードバック

1. **「連続性ステージ」を一級にせよ。** 正準骨格の終端の短い synth では拡散が見つけた分岐の*深掘り*が起きない。**選択された入力群を1 actor に大予算で連結し連続深掘りさせる stage**（mode=fixed count=1＋大 max_tokens＋「方向を再議論せず深掘り」desc）を product 化。発散作業とは**別パス**に分離するのが要（W が示す通り同一文脈では崩れる）。
2. **「立場トーナメント」を documented flow にせよ。** 死角照射は直交*レンズ*（findings-lens-type）、narrow-answer の決定問題は両立しない*立場コミットメント*——と使い分けを明文化。後者は収束を防ぎ overturn を生む。
3. **governed-investigation の product 定義を更新。** narrow-answer では「完全な答え」でなく (a) overturn/死角の発見 (b) 連続性パス用の鋭いブリーフ著者 (c) 監査 が価値。深さは連続性パスへ委譲。
4. **「観点の移植」では代替できないことを明記。** 単一に同じレンズ prompt を1文脈で渡すと劣化する（W）。価値は隔離であり、prompt 注入では再現しない。
5. **「ゲート接地フィルタ」段を持て。** 生成された定量決定ゲートを独立 actor が[接地]/[暫定明示]/[未検証-自信過剰]に分類し、false precision を**根拠導出 or 反証プローブ化に強制変換**する stage。単一は自分の発明した数値を自己検証しない（旗立てゼロ）が、tracefield はこれを段として回せる。実証で未検証-自信過剰が 8→1 に減（theater 検査つき）。relabeling theater を防ぐため再監査でプローブの実測可能性を必ず検査する（findings-lens-type の「フォーマット強制で中身を捏造」回避）。

## 6. 留保（過大主張回避）

- **Y↔Z 差は毎回 run 間分散の範囲内（1〜3点）。** answer スコアの大小での「Y>Z」は*示唆*にとどまる。**だが「単一は安全性/承認/データ共有の順序を独力で先頭化できず、Y がその順序で勝つ」という質的所見は、独立した盲検審判4回すべてで再現**（Y vs X、Y vs Z(web,64k) vs X、Y vs Z(web) vs W、Y vs Z(web off)）。スコア差より、この**反復された質的一致**が頑健な信号。
- **W の最下位（−約7）は比較的大きく、こちらは明確**な信号。
- **web 交絡は小さかった（追試）。** 接戦の Z を web 検索 OFF で再実行しても 43/50 と競争力を維持し（具体性は満点）、依然 Y に順序で敗北。検索ツール呼び出しはゼロ＝公開事実は live 検索でなく**訓練データからの想起**で得ていた。よって§4 で Z に与えた web は結論を歪めていない。
- **「単一の具体性は装飾」説は棄却（反証実験）。** 「単一の高い具体性は固有名・数値の*装飾*では」という仮説を、specificity 軸を**「意思決定を変える具体だけ加点（公開事実・製品名・閾値なき価格レンジは装飾として無視）」に再定義**して再採点して検証。結果は逆で、**単一が 44/50 対 交互織り 40/50 と差を広げた**。監査ログ上、単一が加点された具体は案件選定の除外基準・go/no-go 閾値・原価制約付き価格設計・kill 条件など**装飾でなく決定ゲートそのもの**だった。→ 単一の具体性は本物。
- **よって価値の"種類"が違う、と修正する。** 連続(単一)＝**決定ゲートの広さ**（多数の小さな正しい運用判断を淀みなく量産）。拡散(多エージェント)＝**稀な構造修正の深さ**（単一が滑らせる1つの高ステークスな判断＝安全性/承認の順序を捕まえる）。
- **だが「決定ゲート密度」での単一優位は構造でなくプロンプト抜けだった（再々反証）。** 交互織りの連続性パスに**「決定ゲートを出し尽くせ（選定の数値基準・原価ロジック付き価格・各 pilot の go/no-go 閾値・kill条件・ROI 成功ライン）」を1ブロック明示追加**した Y2 を作り、単一が勝った decision-relevant rubric で再対決→**Y2 が 46/50 対 単一 40/50 と反転勝利**（前回 単一44 対 Y40）。Y2 は approval-first 順序を保持し de-risk=10 を維持したまま、ゲート密度でも単一に並んだ。→ **交互織りは両次元をカバーでき（指示すればゲート密度を出し、かつ単一に出せない構造修正を持つ）、単一は片方しかカバーできない**。「多エージェントの価値は具体性でない」は撤回し、「**指示があれば具体性でも勝てる。単一に対する非対称は"稀な構造修正を独力で出せるか"に局在する**」へ更新。teaching-to-the-test でなく「両次元の和を取れるのは交互織りだけ」という非対称。
- **接地（grounding）でも交互織りがやや上だが modest かつ instructable（反証監査）。** 独立した盲検監査で両計画の定量ゲートを[接地]/[未検証-自信過剰]/[暫定明示]に分類→**単一は 11 個を断定し"未検証"申告ゼロ（純粋な false precision）**、交互織りは 8 個断定だが**3 個を要検証と明示し反証プローブに紐づけ**、監査は交互織りを「より防御可能」と判定。単一は自分の発明した数値を疑わない（整合性バイアスの別の顔）。ただし差は modest（両者とも未検証数値が多い）、旗立ては明示指示由来（instructable）、かつこれは"診断"であって"フィルタ適用"ではない。
- **接地フィルタを"適用"すると false precision は実際に減る（実証・theater 検査つき）。** 監査で[未検証-自信過剰]とされたゲートを「根拠導出 or 暫定＋反証プローブ化」に強制変換するフィルタ段を Y2 に適用→独立再監査（「要検証と書くだけの relabeling theater は[未検証]に落とせ」と指示）で **[未検証-自信過剰] 8→1、[暫定明示] 3→18、[接地] 1→2**。false precision は実質減（theater は1件のみ）。ただし真の[接地]は2件どまり＝多くは「false confidence を honest uncertainty＋実測プローブに変換」であって「数値を第一原理から導出」ではない。単一(Z)はこの自己検証を一切しない（11個を旗立てゼロ）＝tracefield は段として回せるが単一は自己適用しない、が要点。
- 単一モデル系列（codex）・題材1・小 n・審判1（rubric ごと）。一般化には別題材/別モデル系列/複数審判での再現が要る。審判は内容のみ・出自盲検だが、長さ/体裁への暗黙バイアスは完全には排除できていない。
- 題材は守秘のため匿名化した。具体ドメインの記述は本知見の一般性に影響しない（効いているのは構造であって題材内容ではない）。
