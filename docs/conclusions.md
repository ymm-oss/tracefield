# 総括 — 半溶解性オーケストレーション検証プログラムの結論

> tracefield プログラム全体（2026-06-09〜10、実験計画書A → MVP → 本実験 → 攻めの側面）の統合結論。
> 詳細根拠: [`experiment-results.md`](./experiment-results.md)（§1〜14）、[`findings-mvp.md`](./findings-mvp.md)。
> 概念: [`overview.md`](./overview.md) §0。設計: [`design-reference.md`](./design-reference.md) / [`design-agent.md`](./design-agent.md)。

## 0. founding question への回答

実験計画書A §17 の問い:

> 半溶解性オーケストレーションは「よくできたオントロジー」に留まるのか、「統治可能な探索」の設計仮説として成立するのか。

**回答: 設計仮説として成立する ── ただし当初の想定とは異なる形で。**
両翼（守り=統治・攻め=協働）とも機構レベルの肯定的実証に到達したが、その成立条件はどちらも
「構造を作るだけ」では満たされず、**実験によって特定された追加条件**を要した。この条件の発見こそが本プログラムの主成果である。

## 1. 守り（統治可能な探索）— 強く実証

| 主張 | 証拠 |
| --- | --- |
| in-process provenance(C5) は事後再構成(C4)・固定Role(C1/C2) を上回る | C1〜C8 行列: 多段汚染 footprint の recall C5=1.0 / C4=0.75 / C1=C2=0.50 / C3=0（§1） |
| 利得の load-bearing 機構は provenance | ablation C6（C5−provenance）= 0.75 に低下（§2） |
| 「Role を増やせば足りる」は偽 | C2=C1=0.50（§2） |
| 撤回の波及を断てるのは C5 のみ | 隔離 Containment C5=1.0 / C4=0.5、修復(repair)も C5 で成立（§3, Path E） |
| 実データでは C5 の優位は precision/条件スコープ | 採用される汚染B で C5 P=0.50 vs C4 P=0.20（C4 は汚染なし run へ誤帰属常態）（§6f-g） |
| precision 限界（過剰連結）の解 | **ドキュメント接地＋検証可能な引用**: 接地→stance→verify の3層で 0.50→1.00（敵対的引用下でも）（§7） |

**条件の発見（守り）**: C5 の実用ドメインは「**エージェントが独立に疑えない汚染**」（B型=検証不能な権威の証言）に局在する。
反証可能な汚染（A型）は **討議そのものが防御**し（≥2エージェントで無批判採用ゼロ）、追う対象が生じない。
permissive 型（C型）は現行計器では測定不能（§6d-6h）。

## 2. 攻め（深い協働＋偏りの温存）— 条件付きで実証（解錠の物語）

経路は負の結果の連続から肯定に至った:
1. regime 比較・履歴融合では便益が出ず、「完全融合の崩壊」も確率的カスケードに過ぎなかった（§8〜10）。
2. store ベースの用量反応で初の明確なシグナル ── ただし **k_s は均質化ダイヤル**（diversity 0.45→0.08、collapse→0.85）。
   原因洞察: **同一モデルのエージェントは重みレベルで既に完全溶解しており、ペルソナは薄膜**（§11）。
3. 情報異質性（私的文書＋植え込み矛盾）でも発見ゼロ ── ファネル分解で**検索段**（counterpart 非提示）と
   **表現段**（対比しない）の2つの律速を特定（§12〜13）。
4. **解錠（§13〜14）**: 3つのレバーの積層で便益が発現:
   - **手続き k_p**（対比手続きを procedure Entry として配布・採用を citation で記録）→ 初の genuine 発見
   - **serve 多様化**（著者バランス提示）→ 検索段の解除
   - **状況認識**（半溶解の契約をエージェントに明示: 他者の私的知識への唯一の窓・聴衆の存在）→ 表現段の解除＋**均質化の同時解消**

| serve | aware | 発見/3 | collapse | coverage |
| --- | --- | --- | --- | --- |
| similar | なし | 0.33 | 0.51 | 3.7 |
| **diverse** | **あり** | **2.00**（最良 run 3/3） | **0.17** | **8.0** |

**条件の発見（攻め）**: 便益を阻んでいたのはトークン帯域でも同一モデルの限界でもなく、
**(a) 検索の近視、(b) エージェントが協働構造を自覚していないこと**。
→ **半溶解性は「構造 × 契約の自覚」で初めて機能する**。便益と多様性温存は対立せず、自覚の下で同時に改善した（スイートスポット観測）。

## 3. 統一原理

両翼の発見は同型である:

> **半溶解性の機構（store・provenance・手続き）は必要条件にすぎず、
> 「参加者がその契約を知っていること」までを含めて初めて設計が機能する。**
> 守りでは「何が追跡・撤回可能か」の規律（引用の強制・照合）として、
> 攻めでは「他者の状態が何を意味するか」の自覚（situational awareness）として現れた。

また、攻めと守りは**同一基盤上で実装・実証された**（Reference store ＋ Agent ＋ citation provenance）:
発見した便益（横断合成）はそのまま引用で接地され、撤回時には同じ閉包機構で切除できる。
「深く混ざりながら、混ざった痕跡を辿って撤回できる」── 概念（overview §0）の実装が立った。

## 4. 産出された資産

- **アーキテクチャ**: `Tracefield.Reference`（共有状態 store: absorb/serve(:similar|:diverse)/verify/retract+閉包）、
  `Tracefield.Agent`（Jido 2.3 上の facade、private_doc・procedure 採用・aware）、手続き=データ（採用 provenance 付き）。
- **計器**: 命題アンカー判定（IRR 1.0）・埋め込みベース diversity/collapse（決定的）・strict discovery（決定的一次）・
  perception ログ・remeasure（探索と測定の分離）。
- **計器の教訓**: LLM judge は集合レベルへドリフトする（決定的スコアを一次に）／permissive 命題でアンカー判定は破綻／
  クラスタ粒度は本質的に不安定（embedding へ移行）。
- 8 fixture（汚染A/B/C・訂正・デコイ・私的文書3部＋相互作用台帳）、42 tests、実験スクリプト群（`experiments/`）。

## 5. 限界（正直な総括）

- すべて**機構レベルの実証**であり統計的主張ではない（seeds 3〜6・単一シナリオ・gemma4 系のみ）。
- merge/手続き/awareness は **prompt 操作化**（真の重み/メモリ共有ではない。ただし §14 はその水準で便益が出ることを示した）。
- 計器: クロスモデル/人間 IRR 未検証、permissive 汚染は測定不能のまま、事前登録の完全凍結は未。
- awareness プリアンブルは相互作用を示唆しない（公平）が、対比行動全般を促す点で手続きと部分重複の可能性。

## 6. 次段（やるなら）

1. **統計規模**: 最良構成の多 seeds・複数シナリオ・クロスモデル（検出力分析つき事前登録）。
2. **実用パイロット**: アプリ開発文脈（spec/ADR 接地）での Reference＋aware エージェントの実走。
3. **計器**: permissive 汚染の判定法、人間/クロスモデル IRR。
4. 攻めの第2軸: 手続き共有 k_p>1（手続きの伝播・撤回の本実験）。

## 7. 追試（§14 の頑健性確認）

seeds=6（既存3＋新規3、24 runs）で 2×2 を再実行:

| serve | aware | disc_strict/3 | diversity | collapse | ICC | coverage |
| --- | --- | --- | --- | --- | --- | --- |
| similar | 0 | 0.67±0.52 | 0.116 | 0.59 | 2.2 | 3.3 |
| diverse | 0 | 1.50±0.84 | 0.139 | 0.59 | 3.0 | 4.0 |
| similar | 1 | 1.00±0.63 | 0.225 | **0.19** | 4.0 | 7.2 |
| **diverse** | **1** | **1.83±0.75** | 0.204 | 0.22 | **6.2** | **8.5** |

- **両因子の効果は全マージンで同方向に再現**（serve: +0.83/+0.83、aware: +0.33/+0.33）。最良セルは基準の **2.7倍**（~61% の植え込み相互作用を発見、3/3 全発見 run も再出現）。
- n=6 で**役割分担が明確化**: **serve diverse → 発見（検索段）** が主、**aware → 均質化解消・coverage/ICC**（collapse 0.59→0.19-0.22、coverage 3.3→7.2-8.5）が主。両方で全指標最良。
- 基準セル自体も 0.33→0.67 と揺れる（小n のノイズ）が、**順序は全比較で安定** ── §14 の結論は頑健。
