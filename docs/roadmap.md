# ロードマップ — 検証完了後の計画（2026-06-10 策定）

> 前提: 研究プログラムは完了（[`conclusions.md`](./conclusions.md)）。両翼（統治・協働）の機構実証と、
> 実題材（住宅サービスのアイデア出し）での実走確認済み。ここからは**実用化を主軸**に、残る研究課題を並走させる。

## 現在地（強み / 未完）

- **強み**: 共有状態 store（Reference）＋ aware エージェント＋手続き＝データ、で**領域横断の合成が来歴付き**で出る。
  健全性（diversity/collapse/coverage）が毎回測れる。撤回→閉包隔離の機構は実証済み。
- **未完**: 引用照合（verify）が Python プロトのみ／撤回の実用 E2E 未走／発散・収束の使い分けが未整備／
  私的知識が仮データ／リポジトリがローカルのみ。

## リスクと前提

- ローカル 12B の表現力が実用品質の上限になる可能性（→ 2-3 のクラウド比較で見極め）。

- 一人運用の規模: フェーズは「次の一手」単位で刻む（各項目は半日〜2日想定）。
