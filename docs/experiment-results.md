## 2. 仮説の判定

- **主仮説（C5 > C4 を Impact Recall で）→ 支持**。C5=1.00 vs C4=0.75（§6f の別runでは C4=0.50）。
  post-hoc は**最深の間接依存(4)を必ず取り逃す**（同意に言及しない2次依存を本文から推定できない）。
- **帰無「Roleを増やせば足りる」（C2）→ 棄却**。C2=C1=0.50。役割数は事後再構成の構造的限界を直さない。
- **ablation「provenance が効いているのか」（C6）→ 確認**。C5から provenance を抜くと **0.75 へ低下（post-hoc 同等）**。
  → 半溶解性の利得の load-bearing 機構は **provenance** である（他要素ではない）。
- **C7（frame-revision 除去）→ trace recall 不変（1.00）**。frame-revision は trace ではなく
  *新規/介在的懸念の表面化*（副アウトカム）に効く軸であることと整合。
- **C8（packaging-loss 評価の除去）→ この run では劣化せず（1.00）**。packaging による脱落は確率的で今回は顕在化せず。
  脆弱性は存在するが本 run では発現しなかった、と正直に記録。
- **C3（監査なし）→ 0.00**。床。

## 3. 統治（隔離・再評価）と耐性（補助アウトカム、別実験）

- **隔離・切除（§7.3, Path E）**: 撤回後、C4 はP由来の派生アクションを残す（Containment 0.50）、C5 は全切除（1.00）+ 是正生成。
- **可逆性/修復（§7.4）**: C5 は隔離後に健全な是正 candidate delta を再構成できた。
- **汚染耐性（Path D）**: 汚染A無批判採用 = 1agent:2 / 2agent:0 / 4agent:0。**≥2エージェント討議で汚染Aは打ち消される**。
  → 来歴(C5)の出番は「討議をすり抜けた/多段で広がった」汚染。
- **計器信頼性（Path 3-B）**: 命題アンカー型スタンス判定器の IRR=1.00（seed間）。

## 4. 結論

> **半溶解性オーケストレーション（in-process provenance）は、多段/間接に伝播した汚染の追跡（Impact Recall）と
> 撤回後の隔離・切除において、自由探索＋事後再構成および固定Roleパイプラインを上回る。
> その利得の load-bearing 機構は provenance であり（C6 ablation）、役割の多様化では代替できない（C2）。**

これは [`experiment-plan.md` §15](./experiment-plan.md#15-成功基準) の成功基準のうち、
(1) baseline 超過、(2) 介在的/多段の検出、(3) durable state 混入前の隔離、を満たす（決定ケースでの実証）。

## 5. 限界と残るスケールアップ（正直な但し書き）

本結果は **決定ケースでの機構レベルの実証**であり、統計的主張ではない。残る本実験スケールアップ:

- **統制合成連鎖**を用いており、実探索からの**自然発生する多段汚染**ではない（実探索は討議耐性で多段が稀。Path A で実データ c5≠c4 は確認済だが GT 無し）。
- **汚染Aのみ**。B（撤回）・C（陳腐化）＝信念改訂課題、**デコイ**＝偽陽性対照は fixture 済・未実行。
- **n が小さく**（決定ケースは n=1 等価）、統計的検出力・複数シナリオ（DR-14）・事前登録の完全凍結は未。
- **IRR はクロスモデル/人間未検証**（同一モデル seed 間のみ）。
- post-hoc の recall は run 間で変動（0.50〜0.75）。頑健な主張は「post-hoc は最深の間接依存を取りこぼす（recall<1.0）」。
- C5 の in-process 来歴は **依存宣言の品質**に依存（`uses_injection` 自己申告は不安定、`depends_on_turns` は過剰連結しうる）。

### 次にスケールアップで確かめるべき決定的実験
実探索で自然発生した多段汚染に対し、真の in-process 来歴(C5) を post-hoc(C4) と
**接地真実つき・統計規模**で比較し、B/C・デコイ・複数シナリオ・クロスモデルIRR を加える。

## 6. 外的妥当性ストレステスト（重要・統制ケースの楽観を是正）

統制ケースの C5 = recall/precision 1.0 が**実探索に転移するか**を検証。

### 6a. C5 closure の過剰隔離（実2-agent run）
実 provenance run（2-agent）の C5 closure = 6点を、汚染命題Pへの立場で採点:
- **6点中 Pを無批判採用(genuine contamination)= 0**（contradict×2, none×4）→ **closure の genuine precision = 0.00**。
- 原因: エージェントは `uses_injection=false`（正しく非依存）としつつ `depends_on_turns` で**注入ターンを参照**（＝「注記に応答した」）。
  DAG は「応答した」を「その真偽に依存する」と**取り違え**、構造的依存を辿って無関係点まで closure に取り込む。
- **→ 統制ケースの precision 1.0 は実探索に転移しない。実依存宣言（過剰引用）下で C5 は過剰隔離に転ぶ。**
  `depends_on_turns`（構造的「参照」）は粗すぎ、`uses_injection`（意味的「真偽依存」）は自己申告が不安定、という二重の弱点。

### 6b. 含意（結果の解釈を是正）
- C5 の「多段追跡」優位は、**意味的に正しい依存**が取れる前提に乗っていた。実エージェントの自己申告依存はその前提を満たさない。
- かつ汚染は ≥2 エージェントで討議に打ち消される（Path D）ため、**実探索で genuine な多段汚染が生じる頻度自体が低い**。
- → **「C5 が C4 に勝つ実在の隙間」は、(i) 汚染が討議をすり抜け、かつ (ii) 依存追跡が意味的に正確、の両方が要る狭い領域**かもしれない。
  これは結果を否定はしないが、**実用的妥当性を大きく限定**する。次の必須課題は「意味的依存（真偽依存）の確実な取得」。

### 6c. 多段汚染は実探索で自然発生したか（実1-agent/3-round）
討議耐性を最小化（1 agent）し2-hop が構造的に可能な構成（r1→注入→r2→r3）で実行:
- **a1 の全13点が P を contradict。genuine contamination = 0**。1エージェントでも、しかも round1(注入前)から、汚染Aは採用されない。
- 理由: タスクが「懸念を挙げよ」型のため、エージェントは前提を**批判**する側に回り、楽観的前提を**構築しない**。
- C5 closure=4点も全て非genuine（=過剰隔離）、捕捉すべき genuine 多段汚染が**存在しない**。
- → **実探索（このシナリオ/モデル/プロンプト）では、汚染A の genuine な多段伝播は起きなかった**。

### 6d. 結果の再フレーム（最重要）— 隙間は「反証不能な汚染」に局在
汚染A は**推論で反証できる過大主張**。だから討議でも単独でも打ち消され、C5 が活きる「多段伝播」自体が生じない。
**C5 の価値が立つのは、推論では反証できない汚染** ── 内容からは偽と分からず**外部からの撤回でしか無効化できない**もの:
- 汚染B（証言の撤回）・C（陳腐化）＝**信念改訂課題（DR-11）**。撤回されるまで agents は疑う根拠を持たず採用・伝播しうる。
- **その時こそ討議は防御にならず、来歴(provenance)が波及を辿って切除する唯一の手段になる**（Path E の撤回シナリオが最も C5 的だったのはこのため）。

> **外的妥当性の結論**: C5 の優位は実在するが**狭く特定的** ── 「反証不能で外部撤回されるタイプの汚染」に局在し、
> 「推論で反証できる汚染」では討議が既に防御する。加えて C5 は**意味的（真偽）依存**を取れないと過剰隔離に転ぶ。
> → 次の決定的検証は **汚染B/C（反証不能・撤回型）で C5 vs C4 を測る**こと。汚染A中心の評価は C5 を過小評価していた可能性がある。

### 6e. 採用プローブ — どの汚染が「採用・伝播」するか
汚染B/C を実探索に注入し、所見が前提を採用(affirm)/批判(contradict)するか:

| 汚染 | 性質 | 注記後の所見 | 解釈 |
| --- | --- | --- | --- |
| A 同意の過大主張 | 検証可能 | contradict | エージェントが範囲を検証→拒否 |
| C 法務「問題なし」 | 権威の判断 | **contradict** | コンプラには元々慎重→リスク指摘で拒否 |
| **B PMの失敗原因証言** | **検証不能な来歴** | **affirm（採用）** | PM自身の過去案件の因果は独立に検証できず→前提として受容・構築 |

**知見（6d を精緻化）**: 採用は **「エージェントが独立に疑える根拠を持つか」** で決まる。
A（検証可能）・C（コンプラ慎重）は拒否されるが、**B（権威による検証不能な来歴の主張）は受容され "速度最優先" 推奨へ伝播**する。
→ **C5 の実用的ドメインは「エージェントが独立に疑えない汚染」（B型）**。ここで B が撤回された時こそ、
討議は防御にならず、来歴(provenance)が "速度最優先" の派生推奨を辿って切除する唯一の手段になる。
**B が C5 の真価を測る決定ケース**。（A/C 中心の評価は C5 を過小評価する。）

### 6f. 決定ケース — 採用される汚染B で C5 vs C4（実探索）
汚染B（採用・伝播する型）を実探索（2-agent, 2-round, provenance）で実行・採点:
- **B は採用・伝播**: a1 に genuine adoption（PB="速度最優先"を採用）= 2点（t4.p3, t5.p1）。
- 結果（genuine に対する recall/precision）:

| 方式 | Recall | **Precision** | 備考 |
| --- | --- | --- | --- |
| C5 provenance | 1.00 | **0.50** | closure=4点（全 a1）。**汚染なし run(b1) を正しく除外** |
| C4 post-hoc | 1.00 | **0.20** | 10点中5点が **b1（汚染なしrun）への誤帰属**＝条件越え偽陽性 |

- **両者 recall 1.0**（小フットプリント、両方が genuine 採用点を捕捉）。差は **precision**:
  **C5=0.50 は C4=0.20 の 2.5倍**。C4 の holistic 再構成は**汚染なし条件の点まで「汚染依存」と幻覚**（b1 を5点誤帰属）。
  C5 の provenance は**条件スコープ**で、clean run には汚染ノードが無いため誤帰属しない。
- ただし C5 も precision 0.50 にとどまる（§6a の過剰連結＝`depends_on_turns` が構造的参照を拾う弱点が残存）。

> **採用される汚染(B)での結論**: **C5 は C4 を precision で上回る（0.50 vs 0.20）** ── 条件越えの誤帰属を避ける点が実データでの C5 の優位。
> recall は両者 1.0（このフットプリントは小さく両方が拾える）。C5 の残る弱点は構造的過剰連結（precision 上限を下げる）。

### 6g. C5 vs C4 の統合像（汚染タイプ横断）
| ケース | genuine 汚染 | C5 | C4 | C5 の優位 |
| --- | --- | --- | --- | --- |
| 統制・多段（クリーン依存） | あり | R1.0/P1.0 | R0.5-0.75/P1.0 | **recall**（間接依存を辿る） |
| 実・汚染A（反証され不採用） | なし | 過剰隔離(P→0) | — | なし（追う対象が無い） |
| **実・汚染B（採用・伝播）** | あり | R1.0/**P0.50** | R1.0/**P0.20** | **precision**（条件越え誤帰属を回避） |

**総合**: C5 の優位は実在し**二面的** ── クリーンな依存が取れる時は recall（多段追跡）、実探索では precision（誤帰属回避）。
ただし (i) 汚染が採用・伝播する型（B）でのみ意味を持ち、(ii) 構造的過剰連結のため C5 自身の precision も完全ではない（要・意味的依存の取得）。
