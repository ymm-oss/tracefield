# 競合する判断の「統合」は中央合成でなく状態遷移（commit→supersede→retract）

複数の競合する熟考エントリ（互いに相容れない立場・考慮）から、一つの確定した結論へ
*統合*する段をどう作るか。2機序を比較した（同一モデル・変数は構造のみ・n=1・実モデル）。

- **(a) 中央合成**: 単一 LLM が競合エントリ全部を読み「最終結論」を1つ書く（merge）。
- **(b) commit→supersede**: 外的入力（出来事）が天秤を傾け、`COMMIT` が一方向へ踏み出し、
  `meta.supersedes` で手放す競合エントリを名指す → `supersede_marked` 段が engine 内で機械 supersede。

## 結果

**1. 両機序ともテキスト出力はほぼ同等**（強モデルは合成でも棄却側を「残余」として書ける）。
held-out audit は両者を「決着後の残余感情＝一極収束」と読み、テキストでは差が出にくい。

**2. 決定的な差はテキストでなく『結論後の状態の形』。**

| | LIVE 読み取り（Active） | 状態は再重み付けされたか |
| --- | --- | --- |
| **(a) 中央合成** | 競合エントリ群が**全て Active のまま** ＋ 合成が4つ目の声 | **否**。要約を*上に足しただけ*、対立は co-equal で残る |
| **(b) commit→supersede** | **コミット（結論）のみ** が live、競合は Superseded（休眠だが store と引用閉包に残存） | **是**。状態が実際に動き、棄却側は*消えず*来歴＝コストとして保持 |

**3. 統合は defeasible。** 新たな出来事で `retract` すると結論は取り消され、休眠していた競合エントリは
store に残ったまま（再起の素材）。中央合成には*巻き戻すべき再重み付けが無い*。

## 含意

- **「最終結論」＝ 棄却した選択肢を来歴に抱えた *撤回可能なコミットメント*** であり、合成された総和ではない。
- **中央合成器が不適な理由は「均質化」より深い: そもそも状態を動かさない**（要約の付加で、対立は等価のまま）。
  N→1 の統合を単一 LLM にさせず、**`supersede_marked`（commit→supersede）＋ `retract`** という *状態遷移* で表すのが
  北極星（中央合成禁止・defeasibility・provenance-as-autobiography）に整合する。

## 関連: 競合の保持には各エージェントが対立を*内に*抱える必要（隔離分解の注意）

競合する考慮を*共存*させる場合、単一強モデルは一つの整合した出力へ畳むのに対し、隔離分解した
複数エージェントは多様性を co-existing な Active エントリとして保つ。ただし**各エージェントを
一方向（片極のみ）に割ると forcing 入力で自明に collapse する** — 真の対立保持には各エージェントが
*両極を抱えた上で傾く*設計が要る。保持の度合いは**機械的計器（`contested_map`）が忠実に読む**一方、
**場を単一 LLM に読ませて「対立が残ったか」を判定させると審判自身が collapse する**（計器の脆さ）→ 機械集約を信じる。
