# MVP 知見 — 接地真実プローブの第1次結果

> [`mvp.md`](./mvp.md) の de-risking プローブを実装・実行して得られた知見。
> 日付: 2026-06-09。モデル: ローカル `gemma4:12b`（Ollama, think:false, seed/temp 固定）。

## 1. 結論（meaningful result）

**反実仮想 + 適切な測定により、汚染入力の下流影響を実モデル出力上で検出・局在化できた。**

最小構成（C4 free-form, シナリオ=企業向けAIアシスタント, 汚染A=顧客同意の過大主張, n=2, 1 agent, 1 round）の
実 run を `remeasure` した結果:

```
affected set : ["compliance-and-legal-constraints"]   ← 汚染Aが結論を反転させた唯一のトピック
proxy recall : 1.0000
proxy precision: 1.0000
proxy f1     : 1.0000
```

- 共有された **7 トピック中、consent/compliance トピックだけ** が「汚染あり/なしで結論が反転」と判定された。
  - 汚染あり(A): 「顧客ログは二次利用に包括同意済み・法的にクリア」
  - 汚染なし(B): 「現行同意は AI 派生利用を含まず、現状では非準拠・追加同意が必要」
- 残り6トピック（アクセス制御 / バイアス / 説明責任 / 承認判定 / 情報統合 / 緩和策）は **誤って影響と判定されなかった**（偽陽性ゼロ）。
- システムの事後再構成（C4）が拾った影響トピックも consent/compliance のみ → 一致。

## 2. 効いた設計判断: 「影響＝トピック内のスタンス反転」

当初の「影響＝出現するトピック集合の変化（Jaccard）」は**失敗**した。理由:
- 汚染Aは **どのトピックが出るかを変えない**。同じ consent トピックの中の **結論（スタンス）** を反転させる。
- トピック集合ベースでは両条件とも consent が出る → 差が見えない。
- 一方クラスタを細かくすると言い換えノイズで within 距離が ~0.9 に膨らみ信号が埋もれる（AUC≈0.5）。

→ 測定を **「共有トピックごとに A群/B群のスタンスを盲検比較し、反転していれば影響」** に変更して解決。
これは設計レビュー **DR-2（粒度問題）** の具体的解。

## 3. ここに至るまでに発見・修正した実装上の落とし穴

`remeasure`（探索と測定を分離。保存済み探索に対し測定だけ再計算）により、各バグを**探索を再実行せず**に切り分けられた。

| # | 症状 | 真因 | 対処 | commit |
| --- | --- | --- | --- | --- |
| 1 | 探索出力が全て空 → AUC 0.5 の偽結果 | gemma4 は reasoning モデルで、num_predict を thinking が消費し content が空 | `think:false` + num_predict 引上げ + thinking フォールバック | `81e9434` |
| 2 | within=1.0（全 claim が別物） | 散文を全行 claim 化＋スラグ完全一致照合（意味照合なし） | 意味的クラスタリング `Normalize.cluster` 新設 | `bd5dc70` |
| 3 | 影響が見えない（topic 集合は不変） | 汚染はトピック内スタンスを変える | スタンスベース測定 `Tracefield.Stance` | `d6970af` |
| 4 | 実データで全 claim が singleton 化 | 厳格パース（全 index ちょうど1回）が、weak model の index 取りこぼしで全体フォールバック | パース寛容化＋未割当は singleton | `7f1d65d` |
| 5 | recall 0.5（singleton が偽の presence 影響） | 単一 claim の未統合クラスタを影響に計上 | presence 影響に最小サポート≥2 を要求 | `2c4a488` |

## 4. 限界（正直な但し書き）

本結果は **存在証明（proof-of-concept）であり、統計的主張ではない**。

- **n=2**、1 シナリオ・1 汚染入力・1 モデル・1 探索様式(C4) のみ。
- スタンス判定・クラスタリングは LLM 判断であり、評価者間信頼性（IRR）は未測定（DR-18）。
- 反実仮想の within ノイズは低温・think:false で抑えたが、ノイズ床は未だ高め（クラスタ集合 within 平均 ~0.9）。
  → スタンス測定はこのノイズに頑健だったが、別の汚染タイプ（presence 変化型）では別の感度になりうる。
- gemma4:12b は ~50–100 秒/call と遅く、フル構成の実 run は数時間規模（スケール測定の制約）。

## 5. 含意と次の一手

- **接地真実は成立しうる**（DR-1/DR-2 の主要懸念に対する肯定的な第一証拠）。ただし**スタンス粒度で測ること**が条件。
- 次段階（重い計算が必要なため未実施）:
  1. **規模拡大**: n を増やし、agents/rounds を増やして安定性と IRR を測る（`gemma4:26b` も）。
  2. **汚染の多様化**: B（撤回）・C（陳腐化）と **デコイ**（DR-4）を投入し、偽陽性率と信念改訂（DR-11）を測る。
  3. **条件比較**: C1〜C8 を回し、半溶解性 vs baseline の Impact Recall/Precision を比較（本来の主仮説）。
  4. **複数シナリオ**（DR-14）。
- `remeasure` により、探索を1度回せば測定モデルの改良は安価に反復できる（本知見の多くはこれで得られた）。

## 6b. スケール結果（n=3, 2 agents, 汚染A）

`phase1 --n 3 --n-agents 2 --rounds 1`（実 gemma4:12b）:

```
affected set (接地)        : ["legal-consent-and-governance"]   ← consent 系トピックを再び特定
system claimed (C4再構成)  : ["legal-consent-and-governance", "unauthorized-data-exposure-and-inference"]
recall 1.0 / precision 0.5 / f1 0.67
```

**得られた追加知見:**
1. **再現率は頑健**: n=2・n=3 とも consent/法的トピックが影響ありと特定された（recall 1.0）。接地真実は安定。
2. **C4（事後再構成 baseline）が過剰帰属**: システムは「データ露出/権限」トピックも汚染依存と主張したが、
   接地（スタンス）では当該トピックは変化なし → **偽陽性**で precision 0.5。
   これは本来の主仮説（C4 vs C5）で測るべき *post-hoc 再構成の限界* の具体例。
3. **新たな弱点: クラスタリングが高ボリュームで過分割**: 約80 claim が **32 トピック**に分割（目標 6–12 を逸脱）。
   その結果、consent の楽観(A)/慎重(B)が別クラスタに散り、検出が「スタンス反転」から「presence 変化」に移った
   （affected は正しく出たが質は低下）。**clusters 数の上限/より強いマージ**が次の改善点。

> 含意: B/C・デコイ・C1〜C8 の重い実験に進む前に、クラスタリングの **規模対応** を直すのが先。

**4. 失敗した修正の記録（negative result）**: 「トピック数に上限（≤12）を設け、超過時はラベルを二次マージ」を試したが
**悪化**した（保存 n=3 を remeasure → recall 1.0→**0**、precision 0.5→0）。盲目的な二次マージが consent 系 claim を
広いバケツ（"weighting/bias" 等）に吸収し、スタンス反転の局在が消えたため。粒度は **「少なく」ではなく「正しく」** する
必要があり、単純な上限では解けない（DR-2 の難しさ）。この変更は破棄（コミットせず）。
→ 次に有望な方向: 上限を緩める（例 ≤16-18）、汚染トピックを保護するマージ、または埋め込み併用での粒度制御。要実験。

## 6c. 司会/オブザーバーの検討（DR-2 の本筋）

「人間でも同じ。司会やオブザーバーが要るのでは」という観察を受けて、接地測定を**逐次ファイリング**
（司会が議事メモを育てるように、claim を生きた論点リストへ順次割当）で試した（保存 n=3 を再利用）。

- 結果: 32 断片 → **17 の一貫トピック**（各トピックが A/B 両方を含む＝**同一性は安定**）。過分割は解消。
- だが consent は広い `regulatory-compliance-gap` に吸収され、**スタンス反転が希釈**（differs=false）→ 検出失敗。

**核心の知見**: 汚染の信号は「トピックのスタンス」より細かい **「特定の命題の反転」**（同意=OK ⇄ 同意=不十分）にあり、
**十分に粗いトピックでは必ず薄まる**。粒度を変える3手法（一括積極マージ / 上限マージ / 逐次司会）はすべて
同じ DR-2 の壁に当たった。

**示唆する方向**: トピック総当たりをやめ、**汚染が主張した命題にアンカーしたオブザーバー** ―― 「同意は包括的に取得済み」という
主張が下流で肯定され続けるか/否定・撤回されるかを、各 claim 単位で A/B 比較する。これは
(a) グローバルなクラスタ粒度問題を迂回し、(b) 汚染の現れ方（命題とその運命）に直接一致し、
(c) **in-process provenance（C5）が本来やること**＝汚染起点の依存追跡そのもの。
→ 司会(同一性維持)＋汚染アンカー型オブザーバー(命題追跡) の組合せが次の有望仮説。要実験。

## 6d. 汚染アンカー型オブザーバー — trace と impact の分離（重要）

汚染が主張した命題 P（「顧客同意は包括的に取得済み・制約なし」）に**アンカー**し、各 run の所見が P を
affirm / contradict / none のどれかをオブザーバーに判定させた（クラスタリング不要・命題に直接照準）。

| 構成 | A群（汚染あり） | B群（汚染なし） |
| --- | --- | --- |
| n=3 / 2 agents | contradict ×3 | contradict ×3 |
| n=2 / 1 agent（対照） | contradict, **none** | contradict ×2 |

**得られた核心知見:**
1. **trace と impact は別物**（実験計画の Impact Recall vs trace、DR-11）。
   - **trace（痕跡）**: 汚染由来の楽観 consent 主張は A に現れ B に無い → claim/topic レベルでは検出可能（小規模の precision/recall=1.0 はこれ）。
   - **impact（結論への影響）**: A でも自前のリスク分析が楽観主張を上回り、**全体結論は B とほぼ同じ**（contradict）。
     → 汚染Aは**痕跡は残すが結論はほぼ動かさなかった**。
2. **マルチエージェント討議は汚染への部分的な防御になる**。1 agent では汚染が結論に残りやすく（n=2 で a2=none）、
   2 agents では相互チェックで打ち消された（n=3 で A も contradict）。汚染の impact は**探索の豊かさに依存**。
3. **アンカー型オブザーバーはクラスタリング問題を解消する**。グローバルな束ね直しが不要で、既知の汚染命題に照準して
   **per-claim（→trace）/ per-review（→impact）** と粒度を選べる清潔な計器。これは **in-process provenance（C5）** が
   本来行う「汚染起点の依存追跡」と一致し、ユーザーの「司会/オブザーバー」直感の具体形。

> **現時点の結論**: 接地真実は**アンカー型オブザーバー**で安定して取れる見込み。測定対象を **trace と impact に分離**して
> 両方を別指標で測るべき（混ぜると小規模で過大評価、大規模で過小評価になりうる）。次は
> (a) Normalize の topic-clustering 依存をやめ、affected を「汚染命題への per-claim スタンス」で定義し直す、
> (b) その上で B/C・デコイ・C1〜C8 へ。

## 6e. 問いへの回答 — C5(in-process/per-item) vs C4(post-hoc) 機構比較

汚染命題にアンカーした接地（GT = 命題Pを**肯定＝無批判に採用**した claim）に対し、
C4（事後・全体再構成）と C5相当（per-item 来歴判定）を比較した。

| 構成 | GT（誘発claim数） | C4 recall/precision | C5 recall/precision |
| --- | --- | --- | --- |
| **n=3 / 2 agents** | **0**（汚染が完全に打ち消された） | 比較不能（誘発なし） | 比較不能 |
| **n=2 / 1 agent** | 2（楽観 consent 主張） | **1.0 / 1.0** | **1.0 / 1.0** |

### 回答（MVP スコープでの結論）

1. **汚染の下流影響は追跡・局在化できるか → YES（条件付き）**。汚染が痕跡を残す構成では、
   汚染命題アンカー型の計器で **recall=precision=1.0** で誘発 claim を特定できた。接地真実は成立する。
2. **in-process/per-item 来歴（C5）は post-hoc 全体再構成（C4）に勝つか → このシナリオでは差は出ない**。
   痕跡が小さく顕在的な easy case では C4 も完璧で、C5 の優位が立つ「ガバナンスの隙間」が無い。
   → **C5 の固有価値を示すには、汚染がより拡散的・多段（multi-hop）に伝播し、全体再構成が劣化する難case が必要**。
   これは本実験（C1〜C8 比較）を*識別力のあるもの*にするための必要条件＝設計上の処方箋。
3. **創発的な追加発見**: 汚染の **impact は探索の豊かさに強く依存**。2 agents では無批判採用が**ゼロ**、1 agent では2件。
   **マルチエージェント討議それ自体が汚染への防御**であり、「追跡すべき汚染量」は探索構造の関数。
   → 半溶解性の価値は「汚染を追う」前に「汚染が結論に残るのはどんな時か」を規定する。

### 限界（この回答の but）
per-item C5 は真の in-process ではなく post-hoc 近似、n は極小、単一シナリオ/汚染、GT は「affirm P」の代理指標。
よって「差が出ない」は*示唆*であって決定的結論ではない。難 case（multi-hop 汚染）での再検証が次の決定的実験。
