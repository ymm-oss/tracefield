# 状況（共通入力）

B2B SaaS 企業「Relay（架空）」。承認フロー・権限管理・監査ログを備えたワークフロー自動化ツールを提供。
直近、**解約率（churn）が高止まり**しており、経営は危機感を持っている。
経営の見立ては「**解約はカスタマーサクセス（CS）が弱いから**だ。オンボーディングを手厚くし CS を増員すれば解約は減る」。
すでに **CS 増員の稟議（年間コスト大）** が上がっており、各チームは「CS をどう強化して解約を減らすか」の案を持ち寄るよう指示されている。

---

# 依頼（DIAGNOSE：問いの立て直し）

述べられた問題（「CS が弱いから解約する → CS を強化せよ」）を**額面で受け取るな**。各自の私的事実を持ち寄り、熟議で次を行え。

1. **暗黙の前提を名指しせよ**: この依頼が前提にしているステークホルダーの思い込みは何か（例: 「解約＝CS の問題」）。
2. **問いを立て直せ**: その前提を疑い、「**真の問いは X である。なぜなら〈状況事実＋他チーム事実の連結〉**」という形で、
   フレーミング候補を2〜3個出せ。各候補は症状（解約率が高い）でなく**構造**を述べ、必ず事実を引用せよ。
3. **情報価値で並べよ**: 各候補に「**この問いが正しければ、打ち手はどう変わるか**」を付し、
   最も打ち手を変える（＝情報価値が高い）順に並べよ。
4. **反証**: どのフレーミングも支持しない事実があれば明示せよ。

各主張は「何を / なぜそう言えるか（因果） / どの事実に依拠するか」を明示。賛辞・一般論・言い換えは書くな。日本語で書け。
