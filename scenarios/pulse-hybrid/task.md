# 状況（共通入力）

B2C 学習アプリ「Pulse（架空）」。直近3ヶ月で **DAU と週あたり平均セッション数が低下**（約 -18〜22%）。
経営の見立て:「**エンゲージメントが落ちている。利用が減れば解約につながる。危機だ。**
ゲーミフィケーション施策（通知強化・ストリーク・ランキング）で利用頻度を取り戻せ」。すでに**施策パッケージの開発 GO** 寸前。

---

# この熟議の構成（2層）

本熟議には**2種類**のエージェントがいる。

- **データ資料エージェント**（CHANGELOG / ANALYTICS / BILLING / RESEARCH）: 各々が特定の資料そのもの。
  **自資料が支持する事実だけ**を述べ、**沈黙を明示**し、他資料と一致/矛盾を **corroborate / contradict** で示す。
  意見・施策提案はしない（事実を出す層）。
- **意思決定レンズ**（GROWTH_LEAD / VALUE_LEAD）: **データ資料が出した事実*だけ*を根拠**に、自分の任務（目的関数）の
  観点から「**何をすべきか**」を論じ判断する。**事実を作らず、必ず出典 entry を引け。** 優先順位の衝突を明示せよ。

# 問い

1. **（データ層）** 述べられた前提「利用頻度低下＝危機。ゲーミフィケーションで取り戻せ」は、各資料の証拠で支持されるか。
   ソース間の一致・矛盾を突き合わせ、証拠が指す真の構造を述べよ。
2. **（ロジック層）** その確立された事実を踏まえ、**Pulse は何をすべきか**。意思決定レンズが、データ資料の entry を引用して
   判断・優先順位付けし、施策 GO の是非と代替を、トレードオフ込みで結論せよ。

賛辞・一般論・言い換えは書くな。日本語で書け。
