# qwen をプログラミングで活かす — 脱バイアス版の結論

このシナリオ（`qwen-coding-tasks`）で、ローカル中位モデル qwen をソフトウェア開発で有効に使えるタスクをアイデア出しし、**ディベートで篩にかけた**結論。最大の知見は「qwen が*できる*タスク」と「qwen を使う*価値がある*タスク」は違う、ということ。

方法と検証は `docs/findings-bet2-overturn.md`「アイデア出し(拡散)」を参照。射程は **n=1・codex 審判・散文採点 ＝「検証済みの結論」でなく強い仮説**。実運用前に候補を実データで小さく回し、下記 (c) 検証器が本当に採否を返すかを確かめること。

## 判定ルール（ディベートが結晶させた採用条件）

qwen を投入する価値があるのは、次が**すべて**揃うとき:

> **(a)** qwen 固有優位が決定的（オフライン / 機密 / エアギャップ / 極小単価）**または**既存の無料ツールが未カバー
> **＋ (b)** 入出力が閉じている
> **＋ (c)** 検証器（型 / テスト / lint / schema）が採否を返す

1つでも欠けると脱落する — 定番は「(a) 既存ツールで足りる」で、新領域は「(b) 実は閉じていない」で。

## ✅ 本当に勝てる（条件を満たす）

1. **機密・閉域での実行**（frontier が*そもそも使えない*＝限界価値が最大）
   - 院内/社内ルールの静的検査・禁止 API 検出
   - オンプレ社内 SDK の利用例生成
   - 機密設定テンプレ検査（K8s / Terraform / Ansible）
   - エアギャップ依存・ライセンスの棚卸し
   - 秘密値流出スキャンの一次選別（最終判定は既存 secret scanner）
   - 機密リポジトリの命名・ライセンスヘッダ統一
2. **大量・常時の機械的分類 / 正規化**（frontier だとコスト・rate-limit が痛い反復）
   - CI 失敗ログのシグネチャ集約・一次分類
   - フレーク再現マトリクス生成
   - TODO / FIXME 重複束ね
   - i18n 翻訳キー監査（欠落 / 孤立 / 近似重複）
   - 依存更新候補の抽出
3. **自然言語が絡む差分監査**（lint / IDE が拾えない所*だけ*）← ディベートで唯一クリーンに生き残ったニッチ
   - コメントと仕様の不整合
   - レビューコメントの重複集約
   - リリースノートの機密差分チェック
   - ※lint・型・静的解析で機械的に拾える部分は**除外**し、自然言語判断が要る部分に限定するのが条件

## ⚠️ できるが投入する価値は薄い（“定番の罠”）

型アノテーション追加 / テスト雛形 / docstring 生成 / OpenAPI→型・SDK / fixture 量産 / 設定形式変換 / 正規表現変換。

→ **openapi-generator・tsc・schema faker・property-based test・IDE/LSP・lint/formatter が既に*無料で*やる**。qwen を挟むと採用の限界価値ゼロ＋保守負担増。単一 SYNTH も MAGI 多数決も嬉々としてここを選んだが、ディベートが「低価値」と暴いた領域。**qwen 単発のアイデア出しが最初に収束する先＝定番＝罠**。

## ❌ 使うな（frontier 行き）

新規アーキテクチャ設計 / 微妙な正しさ / 広い文脈横断の深い推論 / 不可逆判断（DB マイグレーションの妥当性・互換性方針・ドメイン意味解釈）。「機密だから qwen」と飛びつくと、匿名化の再識別リスクや migration の実行時依存（データ量・ロック・アプリ互換）で破綻する。

## メタな答え: tracefield 自身が最良例

qwen を「単発の賢さ」でなく **足場の中の反復実行器官**として使う（このシナリオの `flow.codex-loop.toml` の scatter⇄deepen ループがまさにそれ）。賢さは構造（隔離・反復・検証）が担い、qwen は小さく忠実な文脈を回す。今回「発散ループが定番を破った」のはこの使い方の実証であり、qwen 活用の本命パターン。

## フロー（このシナリオ内）

- `flow.toml` / `flow.mock.toml` … 単発の拡散（scatter→cluster→synthesis）
- `flow.codex-loop.toml` … 反復（scatter⇄deepen ×3）＋決定論*無落とし*クラスタ＝**生成の定番を破る**
- `flow.codex-debate.toml` … 2極ディベート＋機械集約＝**最終選別で定番の低価値を暴く**（推奨の組織化器）
- `flow.codex-magi.toml` … 3体多数決（反面教師: 同一 prior を「合意」に偽装し新領域を黙殺）
