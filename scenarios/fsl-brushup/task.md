FSL（AIネイティブ形式仕様言語）を次バージョンに向けてブラッシュアップする。

FSL とは: Lark+Z3 の検証器 `fslc`、有界モデル検査(BMC) + k帰納法による無限深度証明、
LLM の write→verify→repair ループ用に常に機械可読 JSON を返す、
business/requirements/spec の3層方言（refinement で連鎖、要件IDが全診断に透過）、
非機能要件は離散時刻SLAまで、Claude Code 用 Agent Skill で AI に仕様を書かせる。

目的: 各自の専門領域の観点から、最も効果の高い改善（言語設計・検証エンジン・AIエージェント運用）を
具体的に提案せよ。既知の課題（DOGFOOD 記録・設計トレードオフ）を踏まえ、
領域をまたぐ相互作用・依存・トレードオフを必ず埋めること。
提案は「何を」「なぜ効くか」「FSL のどの既存事実に依拠するか」を明確にせよ。
