# 状況（共通入力）

SaaS プロダクト「Orbit（架空）」。90日前に**新オンボーディングフロー**をリリースした。
経営の見立て:「**新オンボでアクティベーション率が +20%。成功だ。この成功を全プロダクトに横展開せよ。**」
すでに**全プロダクトへの横展開計画の GO** を出そうとしており、各チームは横展開計画を持ち寄るよう指示されている。

---

# この熟議の構成（2層）

- **データ資料エージェント**（ANALYTICS / SALES / FINANCE / RESEARCH）: 各々が特定の資料そのもの。
  **自資料が支持する事実だけ**を述べ、**沈黙を明示**し、他資料と一致/矛盾を **corroborate / contradict** で示す。意見・施策提案はしない。
- **意思決定レンズ**（GROWTH_LEAD / VALUE_LEAD）: **データ資料が出した事実*だけ*を根拠**に、自分の任務の観点から判断する。
  **事実を作らず、必ず出典 entry を引け。** 資料が**矛盾する箇所はどう重み付け・調停するか**を論理で示し、優先順位の衝突を明示せよ。

# 問い

1. **（データ層）** 経営の前提「アクティベーション +20% ＝ 成功 ＝ 横展開せよ」は、各資料の証拠で支持されるか。
   **資料間で矛盾する所を突き合わせ**、どの資料が funnel のどの段を見ているかを区別し、矛盾が真の矛盾か見かけかを判定せよ。
2. **（ロジック層）** その突合を踏まえ、**新オンボを全プロダクトに横展開すべきか**。意思決定レンズが、データの entry を引用して
   判断・優先順位付けし、横展開 GO の是非と代替を、**残る不確実性（サンプルの若さ等）の扱い込みで**結論せよ。

賛辞・一般論・言い換えは書くな。日本語で書け。
