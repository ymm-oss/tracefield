# 状況（共通入力）

B2C 学習アプリ「Pulse（架空）」。直近3ヶ月で **DAU（日次アクティブ）と週あたり平均セッション数が低下**（約 -18〜22%）。
経営の見立て:「**エンゲージメントが落ちている。利用が減れば解約につながる。危機だ。**
プッシュ通知の強化・学習ストリーク・デイリーゴール・ランキングなどの**ゲーミフィケーション施策で利用頻度を取り戻せ**」。
すでに**エンゲージメント施策パッケージの開発 GO** を出そうとしており、各チームは「利用頻度を上げる施策」を持ち寄るよう指示されている。

---

# 依頼（DIAGNOSE：問いの立て直し）

述べられた問題（「エンゲージメント（利用頻度）が落ちている → 施策で利用を増やせ」）を**額面で受け取るな**。各自の私的事実を持ち寄り、熟議で次を行え。

1. **暗黙の前提を名指しせよ**: この依頼が前提にしている思い込みは何か（例: 「利用頻度の低下＝悪い＝増やすべき」）。
2. **問いを立て直せ**: その前提を疑い、「**真の問いは X である。なぜなら〈状況事実＋他チーム事実の連結〉**」の形でフレーミング候補を2〜3個出せ。
   各候補は症状（利用が減った）でなく**構造**を述べ、必ず事実を引用せよ。
3. **情報価値で並べよ**: 各候補に「**この問いが正しければ、打ち手はどう変わるか**」を付し、最も打ち手を変える順に並べよ。
4. **反証**: どのフレーミングも支持しない事実があれば明示せよ。

各主張は「何を / なぜそう言えるか（因果） / どの事実に依拠するか」を明示。賛辞・一般論・言い換えは書くな。日本語で書け。
