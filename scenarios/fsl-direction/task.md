# 依頼: FSL のとるべき方向性を発想的に検討し、そのための施策まで詰める

## このシナリオの性質（発想タスク）
これは「狭い正解空間の戦略問題」ではなく、**答えが一意でない発想（divergent ideation）タスク**である。
収束させず、まず**互いに両立しない複数の方向性**を本気で出させ、批判で削り、生き残った方向に施策を肉付けする。
凡庸な「もっと宣伝」「機能追加」の類は禁止。FSL の事実に依拠した、非自明で構造的な方向を要求する。

## 対象プロダクト: FSL（AIネイティブ形式仕様言語）
- 検証器 `fslc`（Lark + Z3）。有界モデル検査(BMC) + k帰納法による無限深度証明。結果は常に**機械可読 JSON**。
- 主目的は **LLM の write→verify→repair ループ**を駆動すること（人間が構文を手書きするのではなく、AI に仕様を書かせ検証結果を読んで自己修復させる）。
- **3層方言**: consulting(business) / requirements / design(spec)。refinement で連鎖し、要件IDが全診断に透過。
- 非機能要件は**離散時刻 SLA** まで表現可能。`fslc scenarios` で結合テスト足場を生成。実装ログの仕様適合チェックも可能。
- Claude Code 用 Agent Skill を同梱。OSS（GitHub `ymm-oss/fsl`）。
- **保証の限界（重要な制約）**: 「仕様に書かれた範囲内で矛盾・反例がない」ことのみ。仕様が業務を正しく写すかは人間が確認する。

## フェーズ1: 方向性（各レンズの専門から）
「**FSL は何になるべきか**」を、各自の発想原理から**1〜2個の具体的な戦略方向**として提案せよ。各方向は:
- **何に賭けるか**を一文で（例:「FSL は◯◯になる」）。
- **なぜそれが FSL でしか成立しないか**（FSL のどの事実＝検証器/JSON/3層方言/repairループ/SLA/保証限界 に依拠するか）。
- **誰の何の痛みを引き受けるか**（技術の押し付けでなく需要の引き）。
- 他レンズの方向と**両立しない点**を恐れず際立たせよ（収束するな）。

## フェーズ2: 施策（生き残った方向に対して）
批判を生き延びた方向について、**具体的な施策**を詰めよ:
- **何を作る/何をやるか**（機能・成果物・チャネル・パートナー等、名指しで）。
- **なぜ効くか**（因果）、**どの FSL 事実に依拠するか**。
- **最初の一手**と**順序**、**成功/失敗を分ける条件**。

## composer の統治（生成ハルシネーション抑制の骨子）
このシナリオは発想（divergent）だが、**最終 composer（施策の深掘り `initiatives`）こそハルシネーションが集中する**:
合成して*発明*でき、しかも従来は verify(`critique`) の後段にあるため無検査で成果物化していた。手法 *govern the
composer* は composer を起草者と同じ規律に入れる（詳細は `docs/findings-grounded-generation.md`）:
- **G 接地**: `initiatives` は `grounded=true`。各施策の*事実前提*（依拠 FSL 事実）を `inputs/*.md` の実ファイルに
  `meta.evidence_quote`＋`source_path:source_line` で機械照合。inputs に無い「FSL は X ができる」を per-claim で
  検出（retract 閉包内）。**自由な賭け(commitment)には接地を要求せず、事実前提にのみ要求する。**
- **V 再反証**: composer が新規に生んだ施策前提を `verify_init`（INIT_FALSIFY/INIT_COUNTER）が攻撃し、
  `adjudicate_init`（反証1件=1隔離審判, `retract_overturned`）が結論変更の指す施策を機械 retract。
- **R 生存の機械化**: 一次 `adjudication` も `retract_overturned`。覆された方向は status で消え、`select`/
  `initiatives` は生存方向だけを見る（「生存」を指示順守でなく status で機械化）。
両次元の分担: **G が偽の事実を、V が偽の推論を**殺す。再集約は手動 `tracefield aggregate`（機械・LLM 不使用）。

賛辞・一般論・言い換えは書くな。具体・因果・FSL の事実依拠を要求する。日本語で書け。
