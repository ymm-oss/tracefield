# 関数仕様: `compare_versions(a, b)`

2つのセマンティックバージョン文字列 `a`, `b` を比較し、`a < b` なら `-1`、`a == b`（同一優先順位）なら `0`、`a > b` なら `1` を返す。

## シグネチャ
```python
def compare_versions(a: str, b: str) -> int:   # -1 / 0 / 1
```

## バージョン文字列の形
`MAJOR.MINOR.PATCH[-PRERELEASE][+BUILD]`
- `MAJOR.MINOR.PATCH` は非負整数（入力は well-formed とみなしてよい）。
- `-PRERELEASE`（任意）はドット区切りの識別子列（例: `alpha`, `alpha.1`, `0.3.7`）。
- `+BUILD`（任意）はビルドメタデータ。

## 優先順位（precedence）の規則
1. **MAJOR, MINOR, PATCH を順に*数値として*比較する。** 例: `1.0.10 > 1.0.9`（10 > 9）。文字列比較ではない。
2. **MAJOR.MINOR.PATCH が等しいとき、PRERELEASE を持つ版は、持たない版より*低い*優先順位。** 例: `1.0.0-alpha < 1.0.0`。
3. **両方が PRERELEASE を持つときは、ドット区切りの識別子を左から順に比較する:**
   - 両方が数値の識別子 → **数値として**比較（例: `2 < 10`）。
   - 両方が非数値 → ASCII の**辞書順**で比較。
   - 片方が数値・片方が非数値 → **数値の方が低い**。
   - そこまでの識別子がすべて等しいなら、**識別子の数が少ない方が低い**。例: `1.0.0-alpha < 1.0.0-alpha.1`。
4. **BUILD メタデータ（`+...`）は優先順位の比較で*完全に無視*する。** 例: `1.0.0+build1 == 1.0.0+build2 == 1.0.0`（すべて優先順位は等しい＝`0`）。

## 戻り値
必ず `-1` / `0` / `1` のいずれかの整数。
